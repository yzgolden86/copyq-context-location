# 使用说明

## 功能概述

该命令用于在 CopyQ 搜索结果中，快速回到某条记录在原始历史中的实际位置。

适用于以下场景：

- 搜索后想查看某条记录前后的上下文
- 不想创建额外的上下文 tab
- 希望直接回到原始历史继续浏览

## 工作方式

命令执行时会：

1. 读取当前选中项
2. 回到原始标签页
3. 清除搜索过滤
4. 在完整历史中重新查找当前记录
5. 自动选中找到的记录

## 匹配策略

### 1. 整项精确匹配
优先比较整个 item 的打包数据，而不是只比正文文本。

### 2. 文本回退匹配
如果整项匹配失败，则回退为文本匹配。

### 3. 优先最近项
从最新记录向旧记录方向查找，以提高日常使用中的定位成功率。

## 安装

1. 打开 CopyQ
2. 进入命令管理界面
3. 新建命令
4. 复制 `commands/jump-to-original-history.ini` 中的内容
5. 保存

## 使用步骤

1. 在 CopyQ 中执行搜索
2. 选中某条命中记录
3. 按下快捷键 `Ctrl+Shift+G`
4. 命令会自动回到原始历史并定位该记录

## 已知限制

### 完全相同的重复项
如果多条记录完全相同，包括：

- 文本相同
- MIME 数据相同
- 所有内容都相同

则脚本层无法保证一定定位到你原本点击的那一条。

当前策略是优先选择最近的匹配项。

### 记录已删除
如果原始记录已经被删除或变化，则可能无法重新定位。

## 设计取舍

本方案没有采用 UUID 或底层唯一标识机制，原因包括：

- 避免引入额外复杂度
- 不修改 CopyQ 底层
- 保持命令轻量易维护
- 当前需求已经可以通过内容匹配较好满足

## 推荐使用方式

- 搜索后尽快执行定位
- 适合用于查看上下文
- 不建议把它当作数据库主键级别的精确定位工具
